## Overview

keejLib works with PROS's built-in `MotorGroup` class to control multiple motors as a single unit. Motor groups are essential for mechanisms like intakes, launchers, and lifts where multiple motors should move together.

<Note>
keejLib doesn't provide its own motor group wrapper - it uses PROS `MotorGroup` directly. This guide shows you how to use PROS motor groups with keejLib components.
</Note>

## Creating Motor Groups

Motor groups are created using the PROS API:

```cpp
pros::MotorGroup groupName({port1, port2, port3, ...});
```

### Port Numbers

- **Positive ports (1-21)**: Motor spins in default direction
- **Negative ports (-1 to -21)**: Motor direction is reversed

```cpp
pros::MotorGroup intake({-11, -12});      // Two motors, reversed
pros::MotorGroup flywheel({7, 8});        // Two motors, normal direction
pros::MotorGroup lift({-5, 6, -13});      // Three motors, mixed directions
```

## Common PROS MotorGroup Methods

### move_voltage()

Apply voltage to all motors in the group:

```cpp
intake.move_voltage(12000);  // Full speed forward (12V = 12000mV)
intake.move_voltage(-12000); // Full speed reverse
intake.move_voltage(0);      // Stop
```

### move_velocity()

Set velocity target in RPM:

```cpp
flywheel.move_velocity(600);  // 600 RPM
```

### brake()

Stop motors with specified brake mode:

```cpp
intake.brake();  // Use current brake mode
```

### set_brake_modes()

Set how motors brake when stopped:

```cpp
#include "pros/motors.hpp"

intake.set_brake_modes(pros::E_MOTOR_BRAKE_COAST);
lift.set_brake_modes(pros::E_MOTOR_BRAKE_HOLD);
```

## Using Motor Groups with keejLib

### With DriveTrain

Motor groups are primarily used with keejLib's `DriveTrain` class:

```cpp
pros::MotorGroup leftMotors({-1, -2, -3});
pros::MotorGroup rightMotors({4, 5, 6});

keejLib::DriveTrainConstants dtConsts = {
    .gearRatio = 0.6,
    .cartridge = 600,
    .wheelDia = 3.25
};

keejLib::DriveTrain driveTrain(&leftMotors, &rightMotors, dtConsts);
```

### For Other Mechanisms

For non-drivetrain mechanisms, use PROS motor groups directly:

```cpp
pros::Controller controller(pros::E_CONTROLLER_MASTER);
pros::MotorGroup intake({-11});

void opcontrol() {
    while (true) {
        if (controller.get_digital(pros::E_CONTROLLER_DIGITAL_R1)) {
            intake.move_voltage(12000);
        }
        else if (controller.get_digital(pros::E_CONTROLLER_DIGITAL_R2)) {
            intake.move_voltage(-12000);
        }
        else {
            intake.move_voltage(0);
        }
        
        pros::delay(20);
    }
}
```

## Complete Example

```cpp
#include "main.h"

// Create motor groups
pros::MotorGroup leftDrive({-1, -2, -3});
pros::MotorGroup rightDrive({4, 5, 6});
pros::MotorGroup intake({-11, -12});
pros::MotorGroup flywheel({7, 8});

void initialize() {
    // Configure brake modes
    leftDrive.set_brake_modes(pros::E_MOTOR_BRAKE_COAST);
    rightDrive.set_brake_modes(pros::E_MOTOR_BRAKE_COAST);
    intake.set_brake_modes(pros::E_MOTOR_BRAKE_COAST);
    flywheel.set_brake_modes(pros::E_MOTOR_BRAKE_COAST);
}

void opcontrol() {
    pros::Controller controller(pros::E_CONTROLLER_MASTER);
    
    while (true) {
        // Arcade drive
        int forward = controller.get_analog(pros::E_CONTROLLER_ANALOG_LEFT_Y);
        int turn = controller.get_analog(pros::E_CONTROLLER_ANALOG_RIGHT_X);
        
        leftDrive.move((forward + turn) * 127 / 100);
        rightDrive.move((forward - turn) * 127 / 100);
        
        // Intake control
        if (controller.get_digital(pros::E_CONTROLLER_DIGITAL_R1)) {
            intake.move_voltage(12000);
        }
        else if (controller.get_digital(pros::E_CONTROLLER_DIGITAL_R2)) {
            intake.move_voltage(-12000);
        }
        else {
            intake.move_voltage(0);
        }
        
        pros::delay(20);
    }
}
```
